Produce a decision document (not code) covering at least these three angles as distinct proposals:
1. **Free Unity Asset Store assets** — low-poly / isometric building packs that fit the T004 art
   direction (category color coding, tier/construction-ring mechanics). Must be free-for-commercial-use
   (check license terms explicitly, not just "free download").
2. **Open-source GitHub asset packs** — same fit and license check, sourced from GitHub instead of the
   Asset Store.
3. **Custom-built in Unity, in the spirit of "Fortune City"** — art team builds the buildings directly
   in Unity (primitives/ProBuilder/a simple modular kit) hitting the silhouette/readability bar the
   reference game reaches, adapted to our isometric + category-color system.

"Fortune City" (Fourdesire) is the mobile city-builder the director uses as the visual reference for
"buildings that unmistakably read as buildings" — the exact gap T005 round 1 was flagged for. The source
spec already cites it as a reference.

> **⚠️ IP boundary (non-negotiable):** learn from Fortune City's visual APPROACH — readable isometric
> buildings, how it signals building type — but **do not copy its actual art assets, and do not clone
> its look 1:1.** "똑같이 만든다" here means "match the readability/quality bar," not "reproduce their
> copyrighted assets." The output must be our own cozy-pastel + coquette identity (T004), not a
> Fortune City reskin. Flag this to the director if any option would cross into copying.
