## Brief

Build `LifeTown.Platform` (spec §7.4: asmdef, `UnityEngine` allowed, no economy math) — the layer that
gives the already-built `LifeTown.Economy.Core` (T002, done) a real time source and real persistence on
device. This is the second decision-stable, gate-provable slice per the director's 2026-07-16 direction
to continue with the Platform layer next.

1. **`AndroidMonotonicClock : IMonotonicClock`** (spec §7.3.1) — `SystemClock.elapsedRealtime()` via
   `AndroidJavaClass("android.os.SystemClock")` for `ElapsedMs`, `DateTimeOffset.UtcNow` for `WallMs`.
   `EditorMonotonicClock` and `FakeClock` already exist in Core (T002) — do not duplicate; this task adds
   only the device implementation, which is Platform's because it references `UnityEngine`/`AndroidJavaClass`.
2. **Reboot/clock-tamper detection** (spec §7.3.2): `bootTimeMs = clock.WallMs - clock.ElapsedMs` persisted
   at session start; on resume, recompute and compare — if drift > 5000ms, flag the elapsed time as
   untrustworthy so `LifeTown.Economy.Core`'s existing I6 clamp (already built in T002) has real input to
   clamp against. Implement the detection/flagging in Platform; do not touch Core's clamp logic.
3. **Save-file IO**: load/save `SaveFile` (POCO already defined in Core, T002) to on-device persistent
   storage as JSON. Must be **crash-safe** — write to a temp file and atomically replace, so a crash mid-write
   never corrupts the last good save (this is what week 3-4's "crash/reboot recovery" in the spec's ship
   plan §11.2 depends on).
4. Keep the asmdef boundary from §7.4 intact: **no cap/clamp/curve/settlement arithmetic in Platform** —
   it only supplies time and bytes to Core's pure functions.
