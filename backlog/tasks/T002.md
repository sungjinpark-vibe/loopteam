## Brief
Extend T001's parry core with the rest of touchRPG's P0 input grammar and finish 람팡's full pattern
sheet. The GDD (`docs/spec/00-gdd-v0.3.md`) is the **single source of truth** — read §4.1-4.4, §5.1-5.3,
§6.2-6.3, §7.2 before writing code. Where this brief and the GDD disagree, **the GDD wins**.

This is still P0 (GDD §10: *"터치 패링이 손맛이 있는가"*). **Do not build the 3-phase session
orchestrator or party networking** — those are separate tasks (T003, and P1 respectively). Your job is
the input mechanics and the pattern data, wired into the existing demo scene/driver from T001 so each
pattern is independently triggerable and observable.

### Build
1. **IN-3 회피 존** (§4.1/§6.2): tap inside a blue dodge-zone circle → automatic dash-move out of it.
   Visual: blue circle + a guide line from the character + a clockwise remaining-time gauge inside.
   Reuse the existing `TapPriority` system (T001) — dodge zone sits **above monster part, below parry
   marker** per §4.2's existing order; do not renumber that enum, extend it.
2. **IN-5 차지 공격** (long-press-and-release on the monster body): press-and-hold charges the attack;
   **parry is disabled while charging** (high-commitment, high-risk by design — GDD §4.1). Release fires
   it. This is a real risk/reward tradeoff: while charging, an incoming parry marker cannot be answered.
3. **IN-6 러시** (repeat-tap on a groggy weak-point zone): the **only** input where mashing is rewarded
   (§4.4/§6.2) — everywhere else in the game, mashing is explicitly punished. Visual: large circle on
   the weak part + tap ripples + a progress gauge + a finish flourish on completion. This is a **special
   case that must not leak** — do not make any other input tolerant of mashing.
4. **람팡 P2-P7** (§7.2 table), each a data asset on T001's `MonsterPatternSheet`/`MonsterPatternStep`
   structure — no new pattern needs a code change, only new assets:
   - **P2 꼬리치기** (C-1, parry, ±0.35s, 정박이 아닌 반박자 지연/엇박 — the timing offset itself is
     the difficulty, not a new judgment window)
   - **P3 구르기 돌진** (C-2, IN-3 dodge-zone tap, 1.2s window, zone position randomizes left/right)
   - **P4 볼주머니 페이크** (C-1 variant, parry only the real one; fake dissolves grey just before
     triggering. **MUST**: the tell lives ONLY in monster animation — real = both cheeks symmetrically
     puff, fake = one side only. **Never encode the tell in the marker itself** — a marker-visible tell
     would violate the game's core "watch the monster, not the UI" rule and this monster's entire
     identity per §7.2's own note)
   - **P5 대회전 꼬리** (C-3 relay — ±0.5s each). Relay length scales with party size; **for this solo P0
     slice, use the GDD's own solo substitute (§5.2): a 2-3 tap consecutive sequence**, not real
     multiplayer. Do not build networking to satisfy this pattern.
   - **P6 도토리 비** (C-5, dodge-zone-then-ground-tap, 2.0s cast window, multiple fall points, staged
     small damage)
   - **P7 뒤집힘 그로기** (C-4, triggered by a successful P5, belly rush — the only mash-rewarded window,
     6s duration)
5. **Judgment windows stay weapon/pattern-common** (±0.15 perfect / ±0.35 good, from T001's
   `GameplayConfig`, unchanged in GDD v0.3) — a pattern's *difficulty* comes from rhythm/timing
   structure (엇박, randomized zone position, fake vs real), never from a per-pattern judgment window.
   P5's ±0.5s is the one explicit exception in the GDD table — read it from config, still never
   hardcoded.
6. Extend the demo driver (from T001) so each of P2-P7 can be triggered and observed independently,
   the same way P1 was in T001 — no phase system required yet.
