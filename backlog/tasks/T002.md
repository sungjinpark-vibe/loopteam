## Brief

Build `LifeTown.Economy.Core` — the **pure C# assembly, no `UnityEngine` reference** — that holds the
game's economy and integrity math. This is the decision-stable spine: nothing the director decided
changes it, and because it has no engine dependency it is **fully unit-testable, so the mechanical gate
can actually prove it** (spec §7.4: "what makes 관문 1 able to fail on an integrity regression instead of
being decoration").

Implement, per `lifetown/docs/spec/00-mvp-spec.md`:

1. **`IMonotonicClock`** (§7.3) with `ElapsedMs` — the load-bearing time source. Implement the injectable
   seam: the interface, a `FakeClock` for tests (fast-forward hours in microseconds), and an
   `EditorMonotonicClock` fallback. **Do NOT put the Android JNI implementation here** — that belongs in
   `LifeTown.Platform` (it references UnityEngine). Core only knows the interface.
2. **The economy** (§4, §5.4 I1-I7, §9): EXP/coin payout from a session, building level curve, merge
   rules, and the integrity clamps (I1-I7). Bake in the director's decisions:
   - **D5: Tier1 EXP curve = `[0, 60, 240, 720, 2000]`** (NOT the 07-11 `[0,30,120,360,1000]`).
   - **D9: leisure reward multiplier = ×1.0** — a growth category and a leisure category pay
     identically for equal time. Do not add a leisure penalty.
3. **`SessionRecord`, `SaveFile`, `BuildingInstance`** and the other Core POCOs (§9) — plain data, no
   engine types.
4. **Session commit**: given a `SessionRecord` (with `monotonicElapsedMs`), produce the credited
   EXP/coin after the I1-I7 clamps. Deterministic and pure.

## Acceptance criteria (the 클라이언트팀장 scores these — C1-C5, pass 90)
- [ ] `LifeTown.Economy.Core` compiles as its own asmdef with **zero `UnityEngine` references** (the

- [ ] EditMode tests exist and pass for: the D5 curve (a session does NOT take a building Lv1→Lv5 in one
      sitting), D9 parity (game 3h == reading 3h in payout), and the integrity clamps I1-I7 from §5.4 —

- [ ] No placeholder/stub returns — a clamp that always returns the input is a fake gate (spec §5, §10).
- [ ] Deterministic: same `SessionRecord` in → same credited output, no wall-clock dependence.
